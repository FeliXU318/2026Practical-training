# 商家小程序端前端

基于需求说明书和详细设计文档实现的原生微信小程序商家端前端。当前已接入真实后端接口，接口基础地址配置在 `app.js` 的 `apiBaseUrl`。

## 已覆盖功能

- 商家登录：支持手机号密码登录，以及微信登录后选择商家类型、楼栋并绑定手机号。
- 商品与库存：商品列表、搜索筛选、新增编辑、上下架、库存调整和每日鲜果标记。
- 订单处理：待处理、备货中、待自提、配送中、已完成、售后中状态流转。
- 客户自提核销：扫码或手动输入取货码完成订单。
- 售后反馈：退款、补发、说明结果等处理动作。
- 商户中心：统一查看账号绑定状态，进入商家钱包、数据看板、客户自提核销和售后反馈。
- 商家钱包：查看可提现余额、待结算金额和钱包明细，支持提现至微信零钱。
- 数据看板：作为商户中心功能入口，展示销售额、订单量、热销排行、库存变化、履约效率。

## 文档依据

- 需求说明书：SHOP001-SHOP008、FRU001-FRU008 商家用例。
- 详细设计文档：MerchantProductController、MerchantOrderController。

## 使用方式

1. 使用微信开发者工具导入 `merchant-miniprogram` 目录。
2. AppID 可使用测试号或将 `project.config.json` 中的 `appid` 替换为真实小程序 AppID。
3. 手机号密码登录可自行输入手机号和密码；手机号需符合中国大陆手机号格式，首次登录会按所选商家类型和楼栋自动注册。
4. 微信登录需绑定手机号；开发者工具无法获取真实手机号时，可点击“开发调试模拟绑定”。

## 接口切换

`app.js` 中：

```js
globalData: {
  mockMode: false,
  apiBaseUrl: 'http://10.120.113.18:8080/api'
}
```

将 `mockMode` 改为 `false` 后，页面会自动调用真实 REST 接口。
所有后端路径、请求方式、鉴权和响应解析集中在 `utils/backend-api.js`；如果后端接口定义不同，只需调整该文件中的 `ENDPOINTS`。
登录成功后的 Token 会自动以 `Authorization: Bearer <token>` 发送，商品图片使用 `multipart/form-data` 上传。
开发者工具连接内网 HTTP 地址时，可能需要在“详情 / 本地设置”中开启“不校验合法域名、web-view、TLS 版本以及 HTTPS 证书”。
