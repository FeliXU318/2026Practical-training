# 前端页面说明

## 页面路由表

| 路由 | 页面文件 | 功能 |
| --- | --- | --- |
| `/login` | `src/views/Login.vue` | 管理员登录 |
| `/dashboard` | `src/views/Dashboard.vue` | 数据看板 |
| `/product-audit` | `src/views/ProductAudit.vue` | 商品审核 |
| `/secondhand-audit` | `src/views/SecondHandAudit.vue` | 二手审核 |
| `/orders` | `src/views/OrderManage.vue` | 订单管理 |
| `/delivery` | `src/views/DeliveryManage.vue` | 配送管理 |
| `/phone-card` | `src/views/PhoneCardManage.vue` | 电话卡管理 |
| `/activity` | `src/views/ActivityNotice.vue` | 活动公告 |
| `/complaints` | `src/views/ComplaintManage.vue` | 投诉举报 |

后台页面统一使用 `src/layout/AdminLayout.vue`，未登录访问会由 `src/router/index.js` 路由守卫跳转到 `/login`。

## 页面功能与 API

| 页面 | 功能说明 | 调用 API 函数 |
| --- | --- | --- |
| 登录 | 校验默认账号，保存 `token` 与 `adminInfo` 到 localStorage | `login` |
| 数据看板 | 展示今日订单数、销售额、待审核商品、待处理投诉、图表和待办 | `getDashboardSummary`、`getOrderTrend`、`getTodoList` |
| 商品审核 | 商品名称、类型、状态筛选；详情、通过、驳回、导出 Excel | `getAuditProducts`、`getProductDetail`、`approveProduct`、`rejectProduct` |
| 二手审核 | 名称、分类、状态筛选；图片详情、通过、驳回、导出 Excel | `getSecondHandAuditList`、`getSecondHandDetail`、`approveSecondHand`、`rejectSecondHand` |
| 订单管理 | 订单编号、手机号、类型、状态筛选；详情、状态修改、导出 Excel | `getOrderList`、`getOrderDetail`、`updateOrderStatus` |
| 配送管理 | 配送单号、配送员、状态筛选；分配、更新、异常、导出 Excel | `getDeliveryList`、`assignDelivery`、`updateDeliveryStatus` |
| 电话卡管理 | 编辑宣传标题、说明、防诈骗提示、二维码、外链和状态 | `getPhoneCardConfig`、`updatePhoneCardConfig` |
| 活动公告 | 活动和公告的新增、编辑、启用、停用、删除、导出 Excel | `getActivityList`、`createActivity`、`updateActivity`、`deleteActivity`、`getNoticeList`、`createNotice`、`updateNotice`、`deleteNotice` |
| 投诉举报 | 投诉编号、类型、状态筛选；详情、处理结果提交、导出 Excel | `getComplaintList`、`getComplaintDetail`、`handleComplaint` |

## Excel 导出说明

项目使用 `xlsx` 实现前端导出，通用工具位于 `src/utils/exportExcel.js`。目前支持导出当前查询条件下的全部列表数据，并自动按时间生成文件名。

## Mock 数据说明

当前没有真实后端接口，所有业务 API 文件均返回 `Promise.resolve(Mock 数据)`。审核、状态修改、分配、删除等操作会修改模块内存中的 Mock 数组，刷新浏览器后恢复初始数据。

`src/api/request.js` 已封装 Axios 实例，包含 baseURL、token 请求头、响应错误处理和 401 跳转登录逻辑。各业务 API 文件中保留了真实接口调用形式的注释，例如：

```js
// return request.get('/admin/orders', { params })
```

## 接入真实后端需要修改的位置

1. 在 `.env` 或部署环境中配置 `VITE_API_BASE_URL`，不配置时默认为 `/api`。
2. 将 `src/api/*.js` 中的 `Promise.resolve` Mock 实现替换为注释中的 `request.get/post/put/delete` 调用。
3. 按真实后端字段调整表格字段、筛选参数和详情字段。
4. 如后端分页为服务端分页，需要在列表 API 中传递 `page`、`pageSize`，并使用返回的 `total`。
5. 如 token 名称或响应结构变化，需要同步调整 `src/api/request.js` 和 `src/stores/user.js`。
